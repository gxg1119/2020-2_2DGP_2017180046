1) git에 대한 정리

git은 프로젝트의 어떤 부분도 겹쳐쓰지 않게 프로젝트의 변경을 관리하는 버전관리 소프트웨어이다. git을 쓰는 이유는 다인이 동시에 같은 페이지를 작업할 때 누군가의 작업이 겹쳐지거나, 지워질 것을 방지하기 위함이다. 그렇기 때문에 git은 팀프로젝트에 강하고 버전관리를 해주것에 아주 유용하다. git의 작업의 흐름은 작업 디렉토리에 인덱스를 추가하고 HEAD를 commit하는 흐름으로 진행된다.

git이 하는 일에 대한 주요 명령어

init (깃 저장소를 초기화한다.), commit (인덱스에 추가 된 파일을 확정), push(변경내용 실행), branch(메인 브랜치를 따와 자신만의 버전을 만든다. 후에 "master'에 merge한다), pull(갱신), merge(변경된 사항을 master에 추가, 병합), add(새로운 파일 추가, 저장소에 추가하지는 않는다.), check out (변경 내용을 되돌릴 수 있다. but 인덱스에 추가된 내용과 생성한 파일은 남는다.)

GitHub란?

git을 사용하는 프로젝트를 지원하는 웹호스팅 서비스이다. 가장 인기있는 오픈소스 코드 저장소이다.

GitHub의 특징

1. 저장소에 팀원 초대는 무제한, 2. 개인 저장소는 유료, 3. pull -> request 형식
